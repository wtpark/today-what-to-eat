# V5 이후 발전 과제

## 1. 레시피 필요량·인분·단위 환산

현재는 재료 보유 여부 중심이다. 공공 레시피 데이터에서 1인분 계량을 확보한 뒤 다음을 추가한다.

- 레시피별 필요 수량
- 인분 선택
- g·mL·개·팩·모·단 환산
- 실제 보유량 부족 판정

## 2. 공공데이터 수집 파이프라인

- 공공 레시피 API 수집
- 재료명 정규화
- 필수·핵심·보조·양념 분류
- 조리시간·도구·이미지 권리 검수
- 로컬 시드 생성

## 3. 사용자별 냉장고

현재는 개인 시연용 단일 SQLite DB다. 실제 서비스에서는 로그인 또는 `fridge_id`가 필요하다.

## 4. 냉동 전환 날짜

현재는 보관 위치가 냉동이면 0.35 보정한다. 향후 `frozen_at`을 저장해 냉장 기간과 냉동 기간을 분리한다.

## 5. 식재료 상태 세분화

양배추 등 일부 식재료에서 통째·절단·채썬 상태를 구분하면 관리 구간을 더 정밀하게 계산할 수 있다.

## 6. 레시피 이미지와 UI

- 권리가 명확한 이미지 로컬 저장
- 추천 카드 이미지
- 냉장고 선반 시각화
- 모바일 최적화
- `front/app.py` 페이지·컴포넌트 분리
