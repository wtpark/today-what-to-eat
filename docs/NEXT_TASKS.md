# 후속 발전 과제

## 1. 수량과 인분

레시피별 1인분 필요량, 단위 변환표, 사용자 인분 수를 추가해 실제 보유량이 충분한지 계산한다.

## 2. 사용자 계정

로그인과 사용자별 `fridge_id`를 추가해 여러 사용자가 하나의 DB를 공유하지 않도록 한다.

## 3. 추천 스냅샷

`recommendation_id`와 추천 당시의 `matched_inventory_ids`를 저장하고 조리 완료 시 정확히 검증한다.

## 4. 공공데이터

공공 레시피 API를 수집용으로 사용하고 재료명 정규화·필수/보조/양념 분류 후 로컬 시드로 변환한다.

## 5. 레시피 이미지

이용 범위가 명확한 이미지만 로컬에 저장하고 출처 정보를 함께 관리한다.

## 6. 사용 통계

식재료 소진량, 자주 먹은 계열, 반복 메뉴, 폐기 감소량을 시각화한다.
