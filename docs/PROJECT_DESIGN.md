# 프로젝트 설계 요약 V5

## 목표

저장된 냉장고 재고를 선입선출 원칙으로 관리하고, 현재 취향·직전/최근 식사·보유 양념·조리 조건을 함께 반영해 메뉴를 추천한다.

## 데이터

- 식재료 69개
- 양념 32개
- 활성 레시피 73개
- 모든 식재료는 레시피 또는 대체 규칙에 연결
- 레시피는 이력 보존을 위해 삭제 대신 비활성화

## 주요 흐름

```text
식재료 등록
→ SQLite 저장
→ 소비 우선도 계산
→ 현재 조건 설문
→ FastAPI 추천
→ 그룹별 결과와 점수 근거
→ 조리 완료
→ 재고 차감 및 식사 기록 저장
```

## 추천 결과 그룹

- 선택 음식 계열·식사 형태 완전 일치
- 선택 계열의 다른 식사 형태
- 다른 계열의 선택 식사 형태
- 조건 완화 대체 추천
- 각 그룹에서 정확히 한 단위만 부족한 메뉴

## 정합성 원칙

- 표시기한 경과 및 상태 확인 필요 재료는 자동 추천 제외
- 표시기한이 없으면 식재료별 관리 구간을 이용하되 안전기한으로 표현하지 않음
- 사용자 우선 지정은 메뉴 순위에 실제 반영
- 같은 날짜 구매분은 동일 FIFO
- 간이 대체는 완전 충족으로 과대평가하지 않음
- 실제 재료 사용 없이 식사 완료 기록 불가
- 수량 0 재고는 자동 삭제
