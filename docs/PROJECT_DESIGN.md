# 프로젝트 설계

## 목적

사용자가 식재료를 구매할 때 한 번 등록하고, 이후 저장된 냉장고 재고와 양념장으로 현재 상황에 맞는 메뉴를 추천받는 서비스다.

## 사용자 흐름

```text
식재료 구매 → 식재료 등록 → SQLite 저장
메뉴 필요 → 기본 조건 입력 → FastAPI 추천 → 결과 확인
메뉴 선택 → 재료별 사용량 입력 → 재고 차감 + 식사 기록
```

## 화면

- 홈: 냉장고 콘셉트 2×2 메뉴
- 냉장고 현황: 카드·리스트·정렬·검색·수정·삭제
- 식재료 추가: 빠른 선택·검색·기본 정보·상세 상태
- 메뉴 추천: 기본 조건과 상세 추천 설정
- 내 양념장: 저장형 체크 목록
- 식사 기록: 실제 완료한 메뉴 이력

## 백엔드 책임

- 모든 CRUD
- 소비 우선도 계산
- 추천 후보 필터링과 점수 계산
- JSON 반환
- 조리 완료 시 재고·식사 이력 트랜잭션 처리

Streamlit은 계산 결과를 표시하고 사용자 입력을 수집하며, 핵심 추천 계산은 FastAPI에서 수행한다.

## 데이터 저장

- SQLite DB: `/app/data/today_menu.db`
- Docker volume: `today_menu_data`
- 컨테이너 재빌드 후에도 사용자 데이터 유지
- FastAPI 시작 시 테이블 생성, 마이그레이션, 시드 UPSERT

## 예외 처리

- 냉장고가 비어 있음
- 추천 가능 재료가 없음
- 조건을 만족하는 메뉴가 없음
- 추천 결과가 오래됨
- 사용량 0인 식사 완료
- 재고보다 큰 잔여 수량
- 잘못된 레시피 재료 차감
