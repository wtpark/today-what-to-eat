# V4 변경 사항

## 1. 고아 식재료 정합성 해결

삭제 대신 실제 활용 레시피를 추가했다.

| 식재료 | 연결 레시피 |
|---|---|
| 연어 | 레몬 연어 스테이크 |
| 요거트 | 요거트 양배추 샐러드 |
| 시금치 | 시금치나물, 잡채 |
| 레몬 | 레몬 연어 스테이크, 요거트 양배추 샐러드, 아보카도 에그 토스트 |
| 아보카도 | 아보카도 에그 토스트 |
| 당면 | 잡채 |
| 고구마 | 초간단 군고구마 |

FastAPI 시작 시 고아 식재료가 발견되면 실행을 중단한다. 별도 검사 스크립트도 제공한다.

```bash
python3 scripts/check_orphan_ingredients.py
```

## 2. 홈 화면 세션 유지

기존 `<a href="?page=...">` 링크를 제거하고 Streamlit 버튼과 `nav_to()`를 사용한다. 홈 이동 시 브라우저 전체 새로고침이 발생하지 않아 추천 결과와 세션 상태를 유지한다.

## 3. 식사 완료 요청 검증

`/meals/complete`의 `usage`에 최소 1개 항목을 요구한다.

## 4. 테스트 유지보수

식재료·레시피 개수를 코드에 직접 적지 않고 시드 JSON 길이를 읽어 검증한다. 시드 변경 시 단순 개수 변경 때문에 테스트 코드를 함께 수정할 필요가 없다.

## 5. 냉동 보관 보정

냉동 재료는 표시기한·개봉 경과·FIFO·식품군 민감도 점수에 0.35를 적용한다. 이는 안전성 판정이 아니라 냉장 재료보다 소비 우선순위를 낮추기 위한 서비스 정책이다. 표시기한이 이미 경과한 재료를 추천에서 제외하는 기존 규칙은 유지한다.

## 6. 대체 조리기구 그룹

`tools` 안의 중첩 목록은 하나만 보유해도 충족되는 조건이다.

```json
{
  "tools": [["전자레인지", "냄비"]]
}
```

적용 예:

- 계란찜: 전자레인지 또는 냄비
- 군만두: 프라이팬 또는 에어프라이어
- 고등어구이: 프라이팬 또는 에어프라이어
- 냉동 돈까스: 에어프라이어 또는 프라이팬
- 감자 치즈구이: 에어프라이어 또는 오븐
- 초간단 군고구마: 에어프라이어 또는 전자레인지 또는 오븐
- 레몬 연어 스테이크: 프라이팬 또는 에어프라이어

## 7. UX 잔결함 수정

- 표시기한 사용 체크를 해제하면 날짜 입력창을 숨긴다.
- 식재료 종류는 수정하지 않고 삭제 후 재등록하도록 안내한다.
- 공개 `/demo/load`는 과제 전용이라는 경고를 README에 명시한다.
