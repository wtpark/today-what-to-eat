# 추천 점수 정책

## 1. 점수의 의미

모든 점수는 식품 안전 확률이나 섭취 가능성을 판정하는 값이 아니다. 냉장고에서 어떤 재료를 먼저 사용할지, 어떤 메뉴 후보를 먼저 보여줄지를 결정하기 위한 설명 가능한 정책 점수다.

## 2. 소비 우선도

### 표시기한이 있는 경우

| 남은 기간 | 점수 |
|---|---:|
| 경과 | 자동 추천 제외 |
| 오늘 | 50 |
| 1일 | 45 |
| 2~3일 | 35 |
| 4~7일 | 20 |
| 8~14일 | 10 |
| 15일 이상 | 0 |

### 표시기한이 없는 경우

```text
관리 구간 점수 = 40 × min(구매 후 경과일 / freshness_window_days, 1)
```

`freshness_window_days`는 안전기한이 아니라 프로젝트가 소비 순서를 계산하기 위해 정한 관리 구간이다.

### 개봉 점수

```text
5 + 15 × min(개봉 후 경과일 / opened_window_days, 1)
```

개봉 자체에 5점을 부여하고 시간이 지나면 최대 20점까지 증가한다.

### 선입선출·구매 경과

- 구매 후 14일까지 최대 10점
- 같은 품목의 더 오래된 구매일에 최대 10점 추가
- 같은 날짜에 구매한 묶음은 동일한 FIFO 점수

### 식품군 소비 민감도

`perishability_level` 1~5를 0~10점으로 변환한다.

### 냉동 보정

냉동 보관은 날짜·개봉·FIFO·민감도 점수에 0.35를 곱한다. 이는 냉장 재료보다 냉동 재료의 소비 우선순위를 낮추기 위한 정책이다.

## 4. 재료 매칭

매칭은 JSON 배열 순서와 무관하게 다음 순서로 처리한다.

1. 그룹 내 모든 정확 일치를 먼저 배정
2. 남은 요구조건에 대해 동등 대체만 배정
3. 가능한 비중복 배정 중 충족 조건 수와 품질 합이 가장 높은 조합 선택
4. 하나의 실제 재료를 여러 요구조건에 중복 사용하지 않음
5. 간이 대체는 필수·핵심 조건을 충족한 것으로 계산하지 않음

핵심 선택 재료는 `min_core_count`만큼만 필요하다. 예를 들어 `생크림 또는 우유 중 1개`에서 우유를 보유하면 우유가 정확 일치로 사용되고 생크림 조건은 더 이상 부족으로 계산되지 않는다.

## 5. 필터링

다음은 점수 계산 전에 제외한다.

- 상태 확인 또는 사용자 제외 재료
- 표시기한 경과 재료
- 설정 시간 초과
- 조리기구 불충족
- 직전 식사 강제 제외 조건
- 필수 부족 단위 2개 이상

## 6. 최근 식사 다양성

최근 식사와의 중복 감점은 시간 감쇠를 적용한다.

```text
감점 강도 = 기본 감점 × 2^(-경과일 / 반감기)
```

- 낮음: 반감기 1일
- 보통: 반감기 2일
- 높음: 반감기 4일

동일 메뉴의 감점이 가장 크며, 같은 음식 계열·식사 형태·조리 방식은 더 작은 감점을 적용한다.
