# V5 변경 사항

## 1. 표시기한 입력 UX

식재료 추가 화면의 `포장지 표시기한 입력` 체크박스를 `st.form` 밖으로 이동했다. 체크 즉시 Streamlit이 다시 렌더링되어 날짜 입력란이 나타난다.

## 2. 표시기한 없는 식재료 관리 구간

식재료 마스터에 `freshness_window_days`를 추가했다. 표시기한이 없으면 구매 후 경과일을 관리 구간과 비교해 최대 40점으로 소비 우선도를 보완한다.

이 값은 식품 안전기한이 아니라 앱 내부 소비 순서 관리 기준이다.

## 3. 소비 우선도 보정

- 개봉 즉시 5점 + 경과일에 따라 추가 15점
- 같은 날짜에 구매한 같은 품목은 동일 FIFO 점수
- 사용자 지정 우선 사용은 추천 순위용 점수 100으로 반영
- 냉동 보정 0.35 유지

## 4. 직전·최근 식사

점심 전용 문구를 제거하고 다음으로 변경했다.

```text
직전 식사 음식 계열
직전 식사 형태
직전 식사와 유사한 메뉴 처리
최근 식사 기록 반복 회피
```

회피 조건:

- 상관없음
- 가능하면 피하기
- 같은 음식 계열 제외
- 같은 식사 형태 제외
- 계열 또는 형태 중 하나라도 같으면 제외
- 계열과 형태가 모두 같을 때만 제외

최근 DB 식사 기록과 사용자가 입력한 직전 식사가 동일한 경우 중복 감점을 방지한다.

## 5. 추천 결과 그룹

- 선택 계열·식사 형태 완전 일치
- 선택 계열이지만 다른 식사 형태
- 다른 계열이지만 선택 식사 형태 일치
- 조건을 완화한 대체 메뉴
- 각 그룹의 하나 추가 메뉴

로 분리했다.

## 6. 하나만 더 오분류 수정

핵심 선택 재료가 최소 2개 필요한 레시피는 실제 부족 단위 수를 계산한다. 두 개가 부족한 메뉴는 더 이상 `하나만 더 있으면 가능`에 들어가지 않는다.

## 7. 대체 재료 문맥

대체 규칙을 `equivalent`와 `rough`로 나눴다.

- 동등 대체: 완성 가능 판단에 사용
- 간이 대체: 동일한 재료로 완전히 인정하지 않고 부족 또는 경고로 처리
- 조리 방식 및 식사 형태별 허용·제외 조건 지원

따라서 샐러드에서 토마토 대신 토마토소스를 사용하는 식의 부자연스러운 대체가 차단된다.

## 8. 식사 완료 검증

- `usage` 목록 최소 1개
- 실제 사용량이 0보다 큰 재료 최소 1개
- 프론트에서 사용하지 않음 / 전부 사용 / 일부 사용 선택
- 전부 사용량 0이면 프론트와 백엔드 모두 차단

## 9. 재고 수정 정합성

- 수량 0으로 변경하면 재고 자동 삭제
- 상태를 정상으로 바꾸면 이전 상태 메모 삭제
- 미개봉으로 바꾸면 개봉일 삭제
- 표시기한·개봉일 날짜 검증 유지

## 10. 데이터 및 DB 안정성

- 중복 별칭 검증
- 레시피 `active` 컬럼 도입
- 시드에서 빠진 레시피는 삭제 대신 비활성화
- V4 DB에 새 컬럼 자동 마이그레이션
- 건강 상태 응답에 V5 시드 버전 표시

## 11. 화면 가독성

- 원시 Python 가중치 딕셔너리 제거
- 점수는 화면에서 정수로 표시
- 카드에는 핵심 이유 3개만 노출하고 전체 이유는 펼침 영역으로 이동
- 효율적인 추가 재료에 실제로 열리는 메뉴명 표시
- 추천 제외 사유를 상태 확인, 기한 경과, 사용자 제외, 수량 없음으로 분리
